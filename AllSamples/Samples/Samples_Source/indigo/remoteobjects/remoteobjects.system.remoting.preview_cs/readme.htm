Overview

The "Longhorn" communications subsystem (code-named "Indigo") can be used to create remoting applications. The System.Remoting.Preview sample offers a sneak peak into "Indigo" functionality currently under development.  This sample provides an API wrapper over the PDC drop of "Indigo" that simulates the most current design of System.Remoting.  The sample includes two simple “Hello World” scenarios that preview the use of this wrapper API.  Please note, the design of System.Remoting is still on going and will likely change considerably before the final release.

Building

1.      Open the Longhorn Build Environment console window (click Start, click All Programs, click Microsoft Longhorn SDK, click Open Build Environment Window, and click Longhorn Build Environment).

2.      Navigate to the directory where this readme is located.

3.      Type MSBuild and press ENTER.

Running

Running on a single machine:
To run the sample on a single machine, open two command prompt windows in the System.Remoting.Preview directory.  Within the first command prompt, type remserver to start the server application. Within the second command prompt, type remclient to start the client application.

Running on multiple machines:
To run the sample on multiple machines, perform the following steps:

1.      On the server machine, edit the Server.cs file in the RemServer directory.  Replace "localhost" throughout the file with the server machine name.  Then run the server application.

2.      On the client machine, edit the Client.cs file in the RemClient directory.  In the URIs that represent the server location, replace "localhost" with the server machine name.  Recompile the client and run the client application.
