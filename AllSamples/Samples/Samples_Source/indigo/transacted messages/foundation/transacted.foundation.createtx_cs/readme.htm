Overview

The "Longhorn" communications subsystem is code-named "Indigo".
The CreateTx sample demonstrates how to create a simple transaction. This sample creates a transaction instance in various ways, based on a command line argument the user supplies. No actual transacted work will be done, but the user can choose to commit or abort this empty transaction after it is created.

Key Concepts

Transaction:
Transaction instances can be constructed using the static Create method of the Transaction class as follows.

      tx = Transaction.Create();

Alternatively, you can create a new TransactionManager instance, and have it create a transaction for you, as demonstrated below.

      tm = TransactionManager.Create();                                             
      tx = tm.CreateTransaction();
 

In this sample, there are four ways to create a transaction. Besides using the two ways described above, which create an instance of a transaction with default parameters, you can also supply the description, isolation level and timeout values of the transaction.

Commit or roll back a transaction:
To commit or roll back a transaction, call the respective Commit or Rollback methods of the transaction instance.

        tx.Commit();
        tx.Rollback();

Building

1.      Open the Longhorn Build Environment console window (click Start, click All Programs, click Microsoft Longhorn SDK, click Open Build Environment Window, and click Longhorn Build Environment).

2.      Navigate to the directory where this readme is located.

3.      Type MSBuild and press ENTER.

Running

Run the CreateTx.exe file from the command prompt and supply an integer argument to indicate how you want  the transaction to be created. The following are valid argument options:

1: The transaction will be created using the static Create method of the Transaction class.

2. The transaction will be created using the static Create method of the Transaction class, with parameters supplied..

3. The transaction will be created using the Create method of the TransactionManager instance.

4. The transaction will be created using the Create method of the TransactionManager instance, with parameters supplied.

After a transaction instance is successfully constructed, its relevant information, that is, its identifier, state, and isolation level, are displayed on the screen. You then have a choice to commit or roll back this transaction.
