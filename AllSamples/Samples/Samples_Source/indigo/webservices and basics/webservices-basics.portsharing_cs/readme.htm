Overview

The "Longhorn" communications subsystem is code-named "Indigo".
The TCP Port Sharing sample extends the Request Reply sample by using the TcpTransportListener Windows service to enable sharing of a common TCP port between multiple services. In this case, the client and server share the same TCP port.

Key Concepts

The TcpTransportListener is a Windows service that is installed as part of the “Indigo”. It allows “Indigo” services to share a single TCP port, port 808.  If a “Indigo” Web service is assigned an identity role with the soap:  or soap.tcp: scheme and without a specified TCP port, the service automatically registers itself with the TcpTransportListener.

The only difference between this sample and the Request Reply sample is that in this sample the TCP port is not specified in the identity role of either service.

Building

  1. Open the Longhorn Build Environment console window (click Start, click All Programs, click Microsoft Longhorn SDK, click Open Build Environment Window, and click Longhorn Build Environment).
  2. Navigate to the directory where this readme is located.
  3. Type MSBuild and press ENTER.

Running

Before running this sample, make sure the TcpTransportListener is started. This can be done using the Services MMC snap-in or by typing net start TcpTransportListener at the command prompt.  

Running on a single machine:
To run the sample on a single machine, open a command prompt window in the Server directory and another one in the Client directory. At the server command prompt, type host to start the server application. Then, at the client command prompt , type client <name> to start the client application.

Running on multiple machines:
To run the sample on multiple machines, perform the following steps:

1.      On the server machine, edit the Host.exe.config file in the Server directory. Replace "localhost" with the server machine name in the <identityRole> element. Then run the server application.

2.      On the client machine, edit the Client.exe.config file in the Client directory. Replace "localhost" with the client machine name in the <identityRole> element. 

3.      On the client machine, edit the Client.cs file in the Client directory. In the URI that represents the server location, replace "localhost" with the server machine name. Recompile and run the client application.
