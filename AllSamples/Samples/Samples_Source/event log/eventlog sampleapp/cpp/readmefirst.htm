About Sample App

 

This sample shows a single Application reporting Admin, Operational and Debug trace events.  The events are reported via the existing Eventlog APIs, the Crimson APIs and via Software Tracing macros in that order.

1. Admin Event

SampleApp uses ReportEvent (existing Eventlog API) to write one event to the Application log.  The event is declared in SampleApp.Manifest. 

 

 

<event eventID="991" Legacy="true" base="AdminEvent"

message="$(string.Event.PrinterConnectionFailure)">

<adminInstance level="Error"/>

<template>

<PrinterConnectionFailure>

<Tags>

<Tag>Connectivity</Tag>

<Tag>Availability</Tag>

</Tags>

<PrinterName>%String;</PrinterName>

</PrinterConnectionFailure>

</template>

</event>

 

2. Operational events

SampleApp used EvtReport (new Eventlog/Crimson API) to write two events to an Operational channel.  The Operational channel and the events are declared in SampleApp.Manifest.

 

 

<event eventID="1000" message="$(string.Event.PrinterAdded)" channel="Operational">

<operationalInstance/>

<template>

<PrinterAdded>

<PrinterName>%String;</PrinterName>

</PrinterAdded>

</template>

</event>

 

<event eventID="1001" message="$(string.Event.PrinterRemoved)" channel="Operational">

<operationalInstance/>

<template>

<PrinterRemoved>

<PrinterName>%String;</PrinterName>

</PrinterRemoved>

</template>

</event>

 

3. Debug traces

SampleApp also uses Software tracing macros to report several debug trace events.  It declares the a Diagnostic channel in SampleApp.manifest which captures:

·         Its Trace control GUID

·         Its default logging settings: Logging is Enabled in this case.  Level is set to 5 (which means TRACE_LEVEL_VERBOSE) and flags are set to 15 (0xF in hex).

 

 

<Diagnostic>

<Log>

<Enabled wcm:default="true"/>

</Log>

<Publisher>

<Enabled wcm:default="true"/>

<TraceFlags wcm:default="15"/>

<DebugTraceLevel wcm:default="5"/>

<TraceControlGuid wcm:default="886E2A29-F537-4e04-A57B-A28EA6F2EEAD"/>       

</Publisher>

</Diagnostic>

 

 

How to build it

SampleApp is built using the standard NT build environment.  The following files need to be present on the machine where SampleApp will run:

·         SampleApp.exe

·         SampleApp.man

·         SampleApp.pdb (full symbols  needed for decoding the trace messages)

·         SampleApp.exe.manifest

·         SampleApp.exe.mui

·         register.cmd

·         unregister.cmd

 

Also, traceprt.dll should be copied to %windir%\system32 directory.  Traceprt is available in the IDW directory on any of the release shares.

How to register it

1. First copy the files mentioned above to a local directory

2. Run: register.cmd

 

To unregister, run: unregister.cmd

How to run it

·         First, copy the necessary files mentioned above and run register.cmd

·         run: SampleApp.exe

 

How to view the events

·         Open the New Windows Event Viewer (Right click My Computer, click manage, Expand Services and Applications, Expand Windows Event Viewer)

·         Select the Global\Application channel in the left pane.  In the right pane you will see the list of events in the Application log.  These will contain Event ID 991 from SampleApp

·         Select the Applications\Microsoft.SampleApp\Operational channel in the left pane.  In the right pane you will see the list of events in the SampleApp Operational log.  These will contain Event IDs 1000 and 1001 from SampleApp

·         Disable Diagnostic logging for SampleApp (In the Viewer, right click on Application/Microsoft.SampleApp/Diagnostic channel in the left pane and choose properties, in the properties dialog box uncheck Enable Logging and Click OK)

·         Select the Applications\Microsoft.SampleApp\Diagnostic channel in the left pane.  In the right pane you will see the list of events in the SampleApp Diagnostic log.
